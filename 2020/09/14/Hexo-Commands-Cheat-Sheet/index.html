Hexo Commands

Created2020-09-14|Updated2020-09-14|编程
|Post View:

init（初始化）

这条命令会初始化一个 Hexo 博客网站。如果没有设置指定的 [folder] 文件夹路径，则程序会在当前文件夹下创建网站目录。
这条命令其实进行了如下操作：

  1. 从Github上hexo的仓库中将hexo-theme-starter的内容复制到当前目录下
  2. 使用包管理器安装相应的支持文件：Yam 1，pnpn 或者 npm；如果安装了一种以上的包管理器，则按照列表顺序优先使用靠前的包管理器。

new（新建页面）

新建一个文章页面。如果没有设置 layout 布局参数，Hexo 将会使用默认布局。如果文章标题 title 中存在空格，那么需要在标题两边加上双引号。

参数选项 描述
-p, --path 自定义文章的存储路径
-r, --replace 如果文章标题已存在，则替换现有文章
-s, --slug 伪静态化，自定义文章的URL链接

默认情况下，Hexo 会根据文章的标题设置文件的存储路径。对于网页来说，程序会生成一个带有文章名的文件夹，并在其中生成一个 index.html 文件。使用 --path 选项来 覆盖 默认的路径。

generate（生成静态页面文件）

参数选项 描述
-d, --deploy 生成文件之后直接部署到远端
-w, --watch 查看文件更改记录
-b, --bail 如果在生成文件的过程中有任何无法解决的情况，报错
-f, --force 强制重新生成
-c, --concurrancy 并行生成的最大文件数

publish（发布草稿）

server （启动本地服务器）

启动本地服务器。默认情况下，地址是 http://localhost:4000/

参数选项 描述
-p, --port 自定义端口
-s, --static 只加载静态文件
-l, --log 开启日志。

deploy （配置网站）

$ hexo deploy
参数选项 描述
-g, --generate 配置前先生成文件
